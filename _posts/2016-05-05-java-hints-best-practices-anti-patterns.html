1. Always initialize your Boolean variable 


Boolean isValid = Boolean.FALSE;


Other wise don't be surprised to see later in code 'isValid' will be used to handle 3 states. TRUE, FALSE and NULL.

Ex:

if(user.isValid() != null and user.isValid())
   // do something

In some extreme case some business logic can be associated to the null value (like to check if the object is initialized).

Boolean should have only 2 values.

2. Do not write your own utility methods. More you write more you have to maintain. Please take a look at Apache commons for StringUtils and CollectionUtils

    -https://commons.apache.org/proper/commons-lang/
    -https://commons.apache.org/proper/commons-collections/

3. Don't start with interfaces. Interfaces are over used because some one said 'program for interfaces' is best practice

Interfaces are not required unless you expose a contract ( not implementation)
and if you want to implement strategy more than one implementation of an algorithm/logic

And interfaces should not end with Impl.(credit: checkout this blog from Adam Bien)

If you have <<Algorithm>> -----> AlgorithmImpl, what will you name the second Algorithm?
