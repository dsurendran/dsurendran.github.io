ExceptionMapper is an easier way to handle exceptions in JAX-RS.
Just writing our own ExceptionMapper extending ExceptionMapper<T> will handle the exception globally in a web application.

But for some reasons few exceptions are not handled directly for example SocketTimeOutException needs to be handled as follows 


@Provider
public class SocketTimeOutExceptionMapper implements ExceptionMapper<ProcessingException> {
    @Override
    public Response toResponse(ProcessingException exception) {
        if (!(exception.getCause() instanceof SocketTimeoutException)) 
            throw new RuntimeException(exception);
        return Response.ok("Connection Timeout").build();
    }
}

SocketTimeOutException will be thrown when the client api is trying to access some REST API's. 
Time out values can be configured in ClientConfig as follows




ClientConfig clientConfig = new ClientConfig(); clientConfig.property(ClientProperties.READ_TIMEOUT, 7000);
clientConfig.property(ClientProperties.CONNECT_TIMEOUT, 7000);
Client client = ClientBuilder.newClient(clientConfig);
